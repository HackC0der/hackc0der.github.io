一、Python 的底层工作原理（CPython）

Python 是一种解释型语言，其官方实现 CPython 本身是一个用 C 语言编写的大型程序。当我们运行 Python 脚本时，实际上是这个 C 程序（即 Python 解释器）在读取你的代码并执行。值得注意的是：CPython API 的代码库在不同版本之间可能会有很大的差异，所以虽然基本原理应该保持不变，但实际实现并不总是相同的！所以具体问题要具体分析~

（一）变量和 PyObject

在 CPython 中，一切皆对象。每个变量、整数、字符串、字典等，在底层都是一个 C 语言的结构体（struct）。这些结构体都有一个共同的基础头（header），即 PyObject 结构。

参考：类型对象 — Python 3.10.19 文档

关键结构体定义：

  • PyObject.ob_refcnt：这是类型对象的引用计数，由 PyObject_HEAD_INIT 宏初始化为 1。 请注意对于 静态分配的类型对象，类型的实例 (对象的 ob_type 指回该类型) 不会 被加入引用计数。 但对于 动态分配的类型对象，实例 确实 会被算作引用。
  • PyObject.ob_type：它指向一个 PyTypeObject，类似于 c 语言中的 vtable

当定义一个对象（即使是原始类型），该对象将在内存中以某种形式的结构创建。它通常扩展 PyObject 结构，但如果它是一个涉及长度的对象（例如列表），也可以扩展 PyVarObject。

继承示例：PyListObject

这种设计使得任何 Python 对象都可以被安全地向上转型（cast）为 PyObject* 指针，因为它们的起始部分内存布局是一致的。这为后续的漏洞利用提供了可能性。
一些 Python 类型及其特定结构的例子：

  • 字典有 PyDictObject
  • 元组有 PyTupleObject
  • 字符串有 PyUnicodeObject
  • 浮点数有 PyFloatObject
  • 此外，通常从类型中就能很直接地找到结构体的名称

（二）PyTypeObject

PyObject 中的 ob_type 字段是一个指向 PyTypeObject 结构体的指针。这个 PyTypeObject 可以理解为 对象的“类”或“蓝图”，它定义了该类型对象的通用行为和属性。

PyTypeObject 的作用类似于 C++ 中的虚函数表（vtable）。它包含了一大堆函数指针，这些指针指向了操作该类型对象的特定函数。

参照：类型对象 — Python 3.10.19 文档

示例：str() 函数的底层调用

当你调用 str(my_dict) 时，底层 CPython 并不会有一个独立的 str 函数来处理所有类型。相反，它会：

  1. 获取 my_dict 对应的 PyObject。
  2. 通过 ob_type 找到它的 PyTypeObject。
  3. 在这个 PyTypeObject 中查找名为 tp_str 的函数指针。
  4. 调用该函数指针（例如，对于字典，它指向 dict.__str__ 的具体实现）。
  5. 将该函数的返回值作为 str() 的结果。

你可以假设 str 只是一个独立的函数，分别以 my_str 和 my_dict 作为参数执行。这并不完全正确；与 PyTypeObjects 中的许多其他情况类似，它实际上依赖于 PyTypeObject 中存在的函数。在这种情况下， str 函数实际上将执行以下操作：

  • 前往作为参数指定的 PyObject

  • 从 PyObject 头部定位他们的 PyTypeObject

  • 在所述 PyTypeObject 结构中定位 tp_str 字段，然后执行该字段所指向的地址

  • 将返回值用作更高层级的 str 函数的输出

其他内置函数如 repr()、len() 等，也是通过访问 PyTypeObject 中的相应函数指针（tp_repr, tp_as_sequence->sq_length 等）来实现的。

（二）其他

  • 对象的 id 是一个唯一标识符，代表对象在内存中的地址：id(obj)

    • id 是对象在内存中的唯一地址标识
    • id 在对象生命周期内保持不变
    • is 运算符比较的就是 id
    • id 主要用于对象标识比较，而不是具体的数值

  • ctypes.POINTER 是 ctypes 模块中用于 创建指针类型 的工厂函数。它用于描述和操作 C 语言中的指针。

    • IntPtr = ctypes.POINTER(ctypes.c_int)
    • CharPtr = ctypes.POINTER(ctypes.c_char)
    • VoidPtr = ctypes.POINTER(ctypes.c_void_p)
    • POINTER 创建的是 类型，不是实例
    • 使用 pointer() 或类型转换来创建指针实例

  • ctypes.cast() 是 Python ctypes 模块中的一个重要函数，用于 将指针转换为不同的类型。

    • ctypes.cast(obj, type)
      • obj：要转换的指针对象
      • type：目标指针类型

dicttype.__dict__的地址不是固定的——据大模型解释，这是因为：

  • 字典是可变对象，任何对 dict 类的修改都会导致 __dict__ 变化

五、小结

  • 构造fakeobj，劫持_repr_为system，对象头部覆写.bin/sh，print(fakeobj)时劫持到system('/bin/sh\x00')
  • 当 tp_repr 为空，在满足一些判断条件的情况下，会打印对象所在位置——将该字段置空可泄露地址——可以通过遍历ob_type = ob_type + offset的方式尝试
  • python中一切皆对象，通过指针篡改，可以利用类型混淆，实现泄露等目的
  • 列表、元组对象的ob_item可以尝试劫持；字典对象的ma_keys和ma_values可以尝试劫持；
